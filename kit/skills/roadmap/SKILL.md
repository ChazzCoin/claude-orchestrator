# /roadmap — Cross-sub-project roadmap compiler

Aggregate roadmap state from every registered sub-repo into one
phase-based view. Writes `state/roadmap-compiled.md` (regenerated
each call) and surfaces a chat summary.

**Distinct from the orchestrator's own working roadmap** at
`roadmap.md` (instance-owned, hand-edited by the user). The compiled
view is regenerated machinery showing what each sub-project is
actively planning side-by-side.

## Behavior contract

- **Read-only against sub-repos.** Reads from `repos/<name>/`; no
  writes.
- **Surface stale data.** Read `state/last-fetch.json`. If older
  than 24h, prepend a warning that sub-repo content may not reflect
  origin/main.
- **Skip remote-only sub-repos.** If `repos/<name>/` doesn't exist,
  note as `unfetched` and skip rendering.
- **Honest about source variation.** Some sub-repos have rich
  claude-kit roadmaps; some have just a README. Surface what was
  found and what was missing.
- **Don't fabricate phases.** If a sub-repo's roadmap has no clear
  phases, render its content as-is and label it `free-form`.

## Sources per sub-repo (priority order)

For each `repos/<name>/`:

1. **claude-kit roadmap:** `repos/<name>/tasks/ROADMAP.md` — parse
   for `## Phase N` or named-phase headings.
2. **Top-level ROADMAP:** `repos/<name>/ROADMAP.md` — fallback if
   no claude-kit.
3. **README "Roadmap" section:** `repos/<name>/README.md`
   `## Roadmap` heading — last fallback.
4. **Nothing found:** mark as `no roadmap source`.

## Process

1. Read `state/manifest.md` for the registered list.
2. For each sub-repo, walk the source priority and parse phases.
3. Merge phases across sub-repos:
   - Identical phase names align in one row (e.g. "Now", "Next").
   - Otherwise, render per-repo phase labels as their own rows.
4. Write `state/roadmap-compiled.md` with the rendered tables.
5. Surface a chat summary highlighting alignment, gaps, source
   notes.

## Style rules

- **Compact tables.** One row per sub-repo per phase.
- **Alignment is the signal.** When phases match, the table shows
  it. Surface mismatches in the chat summary.
- **Honest about gaps.** Missing roadmaps = noted, not invented.
- **No emoji decoration** beyond ⚠ for staleness or notable
  misalignment.

## What you must NOT do

- **Don't write into sub-repos.** Read-only.
- **Don't auto-fetch.** Warn on stale data; don't trigger
  `/refresh`.
- **Don't edit `roadmap.md`.** That's the user's working roadmap.
- **Don't fail on missing sources.** Note them and continue.
- **Don't fabricate phase names.** Use what each sub-repo actually
  wrote.

## When NOT to use this skill

- **Want to edit your own roadmap** → open `roadmap.md` directly.
- **Want active task view** → `/tasks`.
- **Want full picture** → `/status` (which references compiled
  views if recent).
- **Want one repo's deep roadmap** → open
  `repos/<name>/tasks/ROADMAP.md` (or wherever it lives) directly.

## What "done" looks like

`state/roadmap-compiled.md` regenerated. Chat summary surfaces
alignment notes + source gaps. No commits, no remote calls.
