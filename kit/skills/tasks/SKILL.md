# /tasks — Cross-sub-project active-task compiler

Aggregate active-task state from every registered sub-repo into one
view. Writes `state/tasks-compiled.md` and surfaces a chat summary.

**Active scope only** — in-progress and next-up. Backlogs are noise
for the macro view; for backlog see `/backlog`.

**Output pattern:** [Pattern 4 — Sprint task board](../../output-catalogue.md#4--sprint-task-board)
adapted for cross-repo (one section per sub-repo with item count and
mini-progress dots; collapsible per-task detail). Chat summary uses
[Pattern 28 — Stats card grid](../../output-catalogue.md#28--stats-card-grid)
lite for active counts + stale flags.

## Behavior contract

- **Read-only.** Read from `repos/<name>/`; no writes into sub-repos.
- **Surface stale data.** Same freshness gate as `/roadmap` — warn
  if `state/last-fetch.json` >24h or absent.
- **Skip remote-only sub-repos.** If `repos/<name>/` absent → mark
  `unfetched`, skip task rendering.
- **Honest about source variation.** Kit-enabled = rich task data.
  Non-kit = branch + PRs only. Render both faithfully.
- **Stale-task flag.** A claude-kit active task whose branch has
  had no commit in ≥7 days → flag with ⚠.

## Sources per sub-repo (priority order)

1. **claude-kit active tasks:** `repos/<name>/tasks/active/*.md` —
   each file is one active task. Parse frontmatter for `id`,
   `title`, `status`, `assignee` (if present).
2. **claude-kit advertisement:** `repos/<name>/.claude/active.md` —
   one-task summary if present.
3. **claude-kit "next up":** `repos/<name>/tasks/backlog/` filtered
   by `status: next` or `priority: next` frontmatter (if claude-kit
   uses such a marker — see `kit/claude-kit-reference.md`).
4. **Non-kit sub-repos:** current branch + open PR titles as the
   only "active work" signal.

## Process

1. Read `state/manifest.md`.
2. For each sub-repo, walk priority and collect:
   - Active tasks (id, title, status, assignee, branch, last
     commit on branch)
   - Current branch on `main` HEAD
   - Open PRs (if cached in `state/sync-status.md` or via fast
     `gh pr list -R` call — prefer cache).
3. Compute stale flag for each active task: branch has no commit
   in ≥7 days.
4. Write `state/tasks-compiled.md` grouped by sub-repo.
5. Surface a chat summary.

## Style rules

- **Compact lists.** One line per task; second line for branch / PR.
- **Stale flag is load-bearing.** Stale = ≥7 days no commit on
  branch. Surface in chat summary, not just in the file.
- **Honest about non-kit gaps.** Don't pretend a non-kit repo has
  task state we can't see. Branch + PRs only.
- **No emoji decoration** beyond ⚠ for staleness.

## What you must NOT do

- **Don't write into sub-repos.**
- **Don't recite full backlogs.** Active scope only.
- **Don't auto-fetch.**
- **Don't fabricate task state.** Non-kit = branch + PRs. That's it.

## When NOT to use this skill

- **Want roadmap view** → `/roadmap`.
- **Want full status** → `/status`.
- **Want depth on one repo's tasks** → open
  `repos/<name>/tasks/active/` directly.
- **Want to draft a task spec for a sub-repo** → that's the
  orchestrator's "task drop" path under `sub-projects.md` —
  separate flow, not this skill.

## What "done" looks like

`state/tasks-compiled.md` regenerated. Chat summary surfaces active
counts + stale flags. No commits.
