# State

Files in this directory are **regenerated**, not hand-edited. They
reflect the current shape of the system pulled from authoritative
sources (git, gh, sub-repo `.claude/active.md` files).

## Contents

- **`manifest.md`** — list of registered sub-repos, their git remotes,
  local paths, roles. Hand-edited at registration time (via `/register`)
  and stable thereafter.
- **`sync-status.md`** — generated by `/sync-check`. Current branches /
  PRs / last-commit per registered sub-repo, plus drift flags against
  active migrations. Don't edit this; regenerate it.

## Why this is its own directory

Two reasons:

1. **Don't conflate authored truth with reflected state.** Authored
   files (contracts, decisions, conventions) live in the world of "what
   we decided." Reflected state lives in "what's currently true."
   Mixing them confuses both.
2. **Regeneration is cheap and authoritative; hand-maintenance rots.**
   `gh` and `git log` are the source of truth for repo state. The
   orchestrator queries them on demand rather than pretending to track
   them.

## Manifest discipline

`manifest.md` is the only file in this directory that's hand-authored —
it lists which sub-repos exist and where. Update it through `/register`
when a new sub-repo joins, or directly when paths change. Everything
else is generated.
