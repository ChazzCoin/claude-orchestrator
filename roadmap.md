# Roadmap

Active strategic direction. You maintain this. The orchestrator reads it
on session start to know what you're steering toward.

Roadmap is *what* and *why*, not *who* and *when*. Per-repo execution
lives in sub-kits.

---

**Discipline.** A roadmap entry should answer:

- What changes in the system when this is done?
- Why now (what made this top of stack)?
- What success looks like (observable, not vibes)?

If an item can't answer those three, it's an open question, not a
roadmap item — move it to `open-questions.md`.
